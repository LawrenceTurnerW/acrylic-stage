# Acrylic Stage — 仕様書

> Palette Project(パレプロ)のアクスタを使った、自動進行型の魔法アイドルRPG。
> ハッカソン7日制作。本仕様書は Claude Code(CLI) への引き渡し用。

---

## 1. プロジェクト概要

### コンセプト
机に置いたアクスタが、魔法アイドルとしてライブ(=戦闘)に出撃する自動進行RPG。プレイヤーの操作は**アクスタの選択と前後入れ替えだけ**。事前準備で4体を選んで前列2/後列2に並べ、ライブが自動進行する中で、ピンチ時にアクスタを物理的に動かして配置を変える。

### 世界観
「ライブ=戦場、ファンの歓声=魔力」。アイドルはライブ中に魔法アイドルへと変身し、ステージに現れたモンスターを魔法で打ち倒す。

### 基本情報
| 項目 | 内容 |
|---|---|
| ジャンル | 自動進行型RPG(オートバトラー) |
| 1プレイ時間 | 3〜5分 |
| 入力デバイス | USBカメラ(ArUcoマーカー読み取り)+ マウス |
| 想定環境 | デスクトップ(Windows / macOS) |
| LLM使用 | **なし**(全ロジックローカル完結) |

## 3. キャラクター

### 3.1 ユニット紹介(VTuber未知の人向け)

Palette Project は3つのユニットに分かれており、それぞれ音楽ジャンルが違う。本ゲームではユニットの音楽性をそのまま魔法系統にマッピングしている。

| ユニット | ジャンル | ゲーム内ロール |
|---|---|---|
| 💘 **RouteHeart**(ルートハート) | 王道アイドル | 支援系(回復・バフ) |
| 👑 **REGALILIA**(レガリリア) | ヘヴィメタル | 火力系(高ダメージアタッカー) |
| 💫 **Sputrip**(スプートリップ) | シティポップ | 技巧系(状態異常・速度操作) |

### 3.2 キャラ一覧(全7名)

ステータスは UI 上は **★5段階** で表示する。星と内部数値の対応:
- 攻撃: ★1=10 / ★2=13 / ★3=16 / ★4=19 / ★5=22
- 防御: ★1=8 / ★2=10 / ★3=12 / ★4=14 / ★5=16
- 素早さ: ★1=10 / ★2=12 / ★3=14 / ★4=17 / ★5=20

**ロール**は「キャラの役割と適性位置を決める根拠」。ステータスの★だけでは「ヒーラー」「デバッファー」などが表現できないので、ロール表記と必殺技を併せて見ることでキャラの個性が分かる構造にする。

| キャラ名 | ユニット | 属性 | ロール | 攻撃 | 防御 | 素早さ | 適性 |
|---|---|---|---|---|---|---|---|
| 七海ロナ | RouteHeart | ✨光 | ヒーラー | ★★ | ★★★ | ★★ | 後列 |
| 藤宮コトハ | RouteHeart | 🎵音 | サポーター | ★★★ | ★★★ | ★★★★ | どちらも |
| 江波キョウカ | REGALILIA | 🔥炎 | アタッカー | ★★★★★ | ★★★★ | ★★ | 前列 |
| 鬼多見アユム | REGALILIA | 🌙闇 | トリックスター | ★★★★ | ★ | ★★★★★ | 前列(ハイリスク) |
| 暁月クララ | Sputrip | ❄️氷 | フィールドコントローラー | ★★★★ | ★★★ | ★★ | 後列 |
| 常磐カナメ | Sputrip | 🌪風 | スピードアタッカー | ★★★ | ★★ | ★★★★★ | どちらも |
| 香鳴ハノン | Sputrip | 💧水 | デバッファー | ★★★ | ★★★ | ★★★ | 後列 |

#### ロールと適性位置の対応(直感ルール)

| ロール | 適性位置 | 理由 |
|---|---|---|
| アタッカー / トリックスター | 前列 | 前線で殴る/撹乱する役。前列補正で火力を出す |
| ヒーラー / サポーター / フィールドコントローラー / デバッファー | 後列 | 支援・状態異常・遠隔魔法系。前線に出る必然性がない |
| スピード系・万能系 | どちらも可 | 機動力で対応 |

#### ステータスバランスの方針
- **前列キャラは「防御★3以上(タンク的)」か「素早さ★4以上(回避特化)」のどちらかで前線適性を説明**
- **後列キャラは防御もそこそこで、後ろから魔法を撃ったり支援する**
- 鬼多見アユムだけは例外で「防御★1 / 素早さ★5 のハイリスク前列」、被弾即危険な紙装甲アサシン枠
- 「回復役・サポート役の強さ」はステータスの星では表現せず、**必殺技の効果**(§5)で個性を出す

> 例:七海ロナの「強さ」は攻撃★2ではなく、必殺技「ハートビート・オーロラ」の全体回復+30/声援+10 にある。編成画面では必殺技カードを大きく表示する。

|---|---|---|---|
| 0 | 七海ロナ | 🦊💙 | 青 |
| 1 | 藤宮コトハ | 🍀💖 | ピンク |
| 2 | 江波キョウカ | 🐣🍁 | 赤 |
| 3 | 鬼多見アユム | 👹🍊 | 紫 |
| 4 | 暁月クララ | 🎠💛 | オレンジ |
| 5 | 常磐カナメ | 🌺🌸 | 緑 |
| 6 | 香鳴ハノン | 🎀🎶 | 水色 |

#### カラーの使い分け方針

- **公式個人カラー**:キャラカード、ステータス枠、キャラ名表示などUI全体で使用。各キャラの "色" として一貫させる
- **属性カラー**:必殺技発動時のフルスクリーンエフェクト、通常攻撃の光ラインなど、戦闘演出のみで使用

→ 「カードを見れば誰か分かる(個人カラー)」「技が出れば何属性か分かる(属性カラー)」の二段構え。

### 3.4 アプリ内キャラ紹介UI

VTuber を知らない人もハッカソン会場で触ることを想定し、以下の方針で実装する。

- **編成画面**: アクスタを置くと、画面上にそのキャラの「キャラ紹介パネル」が表示される

  - ロールと必殺技を併せて見せることで「このキャラは何の役割か」が1秒で分かる
  - 編成画面で全7体のパネルを並べて見ることもできる(独立した「図鑑画面」は作らず、編成画面で兼ねる)
- **戦闘画面**: 各キャラのステータスバー横に小さくユニットマーク(💘👑💫)と属性アイコン(✨🎵🔥🌙❄️🌪💧)を常時表示

### 3.5 物理レイアウト・ArUcoマーカー仕様

#### アクスタの向き
**アクスタは正面をプレイヤー側、背面をカメラ側に向けて配置する**。

```
       [PC画面 + カメラ]
              ↑
       前列 [アクスタ正面] [アクスタ正面]    ← プレイヤー側に正面
       後列 [アクスタ正面] [アクスタ正面]
              ↓ (カメラには背面が映る = マーカー認識)
        [プレイヤー]
```

- **プレイヤーから見ると推しの顔が見える**(正面)
- **カメラからは台座背面のマーカーを認識**(美観を損なわない)
- **構図解釈**: 机の上のアクスタが、プレイヤー(=ファン)に向かってライブパフォーマンスを披露している。背景の画面上で敵を撃退する、プロデューサー / 観客視点

#### ArUcoマーカー仕様
| 項目 | 値 |
|---|---|
| マーカー辞書 | `DICT_4X4_50` |
| マーカーID | 0〜6 を7体に割り当て(§3.3 参照) |
| サイズ | 2cm 角 |
| 貼付位置 | アクスタ台座の背面、もしくは本体の背面下部 |
| 印刷方法 | プリンタで黒インク印刷 → 厚紙に貼る → 両面テープでアクスタに固定 |

#### カメラ設定
| 項目 | 値 |
|---|---|
| 解像度 | 1280×720 |
| 認識頻度 | **5fps** |
| 配置 | PC上部のカメラ(内蔵 or USB) |
| 角度 | 机上のアクスタを上から30〜45度の角度で見下ろす |
| 認識範囲 | カメラから30〜60cm程度の机上エリア |

#### 前後判定
- ArUco検出時のマーカーの **Y座標(画面上の位置)** で判定
- 「画面下に近い = 前列」「画面上に近い = 後列」
- **閾値はキャリブレーション可能**にする(設定画面で「ここから前列」のラインを引く)

#### キャリブレーション画面(Day 1-2 で実装)
- カメラ画像を表示し、「前列ライン」を画面に重ねて見せる
- 7体全部を置いて認識テストできるデバッグ画面
- 起動時に必ず通る or 「設定」メニューから開ける

## 4. ゲームルール

### 4.1 基本パラメータ
- **テンション(HP)**: 全キャラ 100 でスタート
- **声援ゲージ(必殺ゲージ)**: 0〜100、満タンで次回行動時に必殺発動
- **ターン進行**: 5秒ごとに1ターン経過(自動)
- **行動順**: 素早さの高い順
- **声援獲得**:
  - 通常攻撃時 +20
  - 被弾時 +10
  - ターン経過時 +5
  - **max_gauge はキャラ毎に上書き可能**(ヒーラー/サポーターは 80-90 で早撃ち、アタッカー系は 105-115 で溜め長め)。これにより全員同タイミング発動を避ける

### 4.2 配置システム
机の上に置いたアクスタを、カメラのY座標で前列/後列に判定。

| 位置 | 被弾率倍率 | 適性位置のとき | 不適性のとき |
|---|---|---|---|
| **前列** | ×1.5 | 攻撃 +20% | 攻撃 -15% |
| **後列** | ×0.5 | 攻撃 +20% | 攻撃 -15% |

戦闘中、アクスタを物理的に動かすと配置が変わり、補正値も即時反映される。

### 4.3 ダメージ計算
```
基礎ダメージ = (攻撃力 × 配置補正 × コンディション補正 × 必殺倍率) - 対象の防御力
最終ダメージ = max(基礎ダメージ, 1)
```
最低保証1ダメージ。属性相性は実装しない(シンプル化のため)。

### 4.4 コンディションシステム
**起動時に各キャラのコンディションをランダム決定**し、そのセッション中は固定。アプリ再起動で再抽選。

| 状態 | 確率 | 効果 |
|---|---|---|
| 絶好調 ✨ | 10% | 攻撃+30%、声援ゲージ蓄積 1.5倍 |
| 好調 | 25% | 攻撃+10% |
| 普通 | 30% | 補正なし |
| 緊張気味 | 25% | 攻撃-10% |
| 寝不足 😪 | 10% | 攻撃-15%、10%の確率で行動スキップ |

### 4.5 戦闘不能・勝敗
- テンションが0になったキャラはダウン、戦闘から離脱
- 味方全員ダウン → 敗北
- 全ての敵を撃破 → 勝利

### 4.6 警告攻撃システム(ボス専用ギミック)★

ボス敵は通常攻撃に加え、「**2ターン後に前列大ダメージ**」を予告してくる。プレイヤーはアクスタを物理的に後列へ動かして前列を空にし、被弾を回避する。

#### 仕様
| 項目 | 値 |
|---|---|
| 抽選タイミング | ボスの行動ターン毎 |
| 発動確率 | 25% |
| 警告から発動までのターン数 | 2ターン |
| 種類 | 前列爆撃のみ(後列狙撃は物理操作上の都合で廃止 — カメラ画角の都合で後列の方が空間に余裕があるため、「全員後列に避難」だけで対応できる前列爆撃に統一) |
| ダメージ | 前列全員に 50固定(防御で軽減) |
| 警告中の連続抽選 | しない(重複防止) |
| セーフ条件 | 前列の生存数が **`safe_threshold` 体以下** ならセーフ(SPEC オリジナルは 0、現状デフォルトは 1)。4体を片側に詰める物理操作の負担を緩和するため緩和した。stages.yaml の `warning_attack.safe_threshold` で調整可能 |

#### 表現の流れ
1. **警告開始**:画面上部に赤い帯「⚠ 2ターン後:前列爆撃!」
2. **1ターン後**:「⚠ 残り1ターン!」点滅 → プレイヤーがアクスタを後列へ動かす ★物理操作のハイライト
3. **発動時**:全画面エフェクト「💥 前列爆撃!」
   - 前列に `safe_threshold + 1` 体以上残ってる → 大ダメージ + 赤画面フラッシュ(前列全員に着弾)
   - 前列が `safe_threshold` 体以下 → 「ヒュー...セーフ!」 + 観客の歓声SE

#### 設計意図
- ルールが「2ターン後に前列にダメージ、後列に避難すればセーフ」の1行で説明できる
- 1ステージで1〜2回発生する想定(発生確率25% × 6〜8ターン)
- 動かさなくても勝てる可能性は残るが、動かした方が圧倒的に楽というバランス
- 段階的なカウントダウンで判断時間を確保(忙しすぎない)
- 後列の方が物理空間に余裕があるため、避難方向を後列に統一して物理操作の難度を下げる

## 7. 戦闘実況テキスト

### キャラ性の濃度(「中」レベル)

| キャラ(ユニット) | 喋り方の方向 |
|---|---|
| 七海ロナ(RouteHeart) | 王道アイドル、明るく前向き |
| 藤宮コトハ(RouteHeart) | 王道アイドル、優しく可愛らしい |
| 江波キョウカ(REGALILIA) | メタル系、熱血で激しめ |
| 鬼多見アユム(REGALILIA) | メタル系、クールで挑発的 |
| 暁月クララ(Sputrip) | シティポップ系、洗練されてアンニュイ |
| 常磐カナメ(Sputrip) | シティポップ系、しなやかでミステリアス |
| 香鳴ハノン(Sputrip) | シティポップ系、ムーディーで艶っぽい |

### セリフ例(イメージ)

## 12. 設計方針 / 遵守事項

### 安定動作優先
- **LLM API は使用しない**(全ロジックローカル完結、ネットワーク不調リスクなし)
- 設定はすべて YAML で外出し、コード変更なしで調整可能に
- WebSocket 断時は再接続を試みる
- **ArUco 検出は 5fps で実行**(60fps は不要、CPU/カメラ/WebSocket への負荷を抑える)。配置変更の反応速度として体感問題なし

### キャラ表現
- **キャラクター画像は各自で用意**(本リポジトリには含めない)
- セリフはユニットの音楽性に寄せた「中」レベルの味付け(本人の固有口癖・方言・特定ネタは入れない、§7参照)
- 技名・敵名・ステージ名はすべてオリジナル(楽曲名は使用しない)

### 拡張性
- キャラ追加・ステージ追加は YAML 編集だけで可能な構造に
- 必殺技の効果は「効果タイプ + パラメータ」のデータ駆動で実装(ハードコードしない)

### 発表時の表記
- 「Palette Project を題材にしたファン制作物」「公式とは無関係」を README とデモ内に明記
- 二次創作ガイドラインを遵守
